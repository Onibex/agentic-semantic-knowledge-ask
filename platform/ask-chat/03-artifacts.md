# ASK Chat · Artifacts

> **Flow 3 of the ASK Chat manual.** Generate shareable business documents — reports,
> executive briefs, and data tables — from your governed semantic layer without writing SQL.
> View, download, edit, and regenerate them at any time.

| | |
|---|---|
| **Who** | Business user / analyst |
| **Time** | ~3 minutes per artifact |
| **Prerequisites** | A workspace is selected, an environment chosen, and a mode set (see [Flow 1](01-workspace-environment-mode.md)). |
| **You'll end with** | A downloadable business document built from live data, saved in your browser for later review. |

**Where this fits:** Configure → Author → Publish → Set controls → Chat → **Artifacts (you are here)**

> The screenshots and sample values below use an illustrative **SAP Sales & Distribution**
> example (Sales Orders). Substitute your own Data Products — the exact demo names and
> questions won't exist in your system.

---

## Concepts (30-second version)

- An **artifact** is a structured business document generated by the agent from your data.
  It combines a written narrative (`content`) with one or more SQL result sets (`datasets`).
- You describe what you need in a short **conversational wizard** — no form fields, no SQL.
  The agent generates the document and data in one call.
- Artifacts are stored **locally in your browser** (`localStorage`). They persist across
  page refreshes but are browser- and device-specific.
- You can **regenerate** an artifact at any time (re-runs the original request), or open the
  **Edit panel** to change the purpose, data focus, or even override the SQL before
  regenerating.

## 1. The Gallery

Open the **Artifacts** page from the sidebar. The gallery shows all saved artifacts as cards
in a responsive grid.

**Toolbar controls:**

| Control | Behaviour |
|---|---|
| **Type filter** (dropdown) | Filter the gallery by document type: All types, Sales Report, Inventory Report, Executive Summary, Financial Report, Custom. |
| **Search bar** | Filters artifact names as you type. |
| **New artifact** button | Opens the creation wizard. |

Clicking any card opens its **Viewer**. Hovering reveals a subtle gradient and a quick-open
indicator.

---

## 2. Create an artifact — the wizard

Click **New artifact**. The page switches to a **conversational wizard** that walks you
through four steps, one at a time. Each step shows an assistant prompt; you type your answer
and press **Enter** to advance.

| Step | Prompt | What to enter | Demo value |
|---|---|---|---|
| **1 · Name** | *What should we call this artifact?* | A short, descriptive name. | `Q1 Sales Performance by Material` |
| **2 · Purpose / audience** | *Who is this for and what decision should it support?* | One or two sentences. | `For the regional sales manager — to review which materials drove revenue in Q1 and where to focus Q2 efforts.` |
| **3 · Data to include** | *What data should be included?* | Describe in plain business terms. | `Net sales value and quantity sold by material for Q1 of the current year, top 10 only.` |
| **4 · Format** | *What format works best?* | Pick a format chip or type your own. | Click **Detailed Report** |

**Format quick-pick chips** on step 4:

| Chip | What it produces |
|---|---|
| **Detailed Report** | A full narrative with sections, analysis, and supporting data tables. |
| **Executive Brief** | A concise summary — key findings, implications, one paragraph per point. |
| **Data Tables** | Minimal narrative; emphasis on structured tables and raw data. |
| **Proposal Format** | A recommendation-oriented document with rationale and next steps. |

After step 4 you see a **summary card** confirming the four values. Click **Generate** to
submit. While the backend works, the wizard shows a rotating **thinking indicator** with
status messages.

> **Tip —** Be specific on step 3. "Top 10 materials by net value in Q1" gives the agent a
> clearer scope than "sales data". The more precise the description, the more focused the
> generated document and datasets.

## 3. View an artifact

Once generation completes — or when you click a card in the gallery — the **Artifact
Viewer** opens.

### Header bar

| Control | What it does |
|---|---|
| **← Back** | Returns to the gallery. |
| Artifact name | The name you gave in step 1. |
| Format badge | Shows the document type (e.g. *Detailed Report*). |
| Token count | Total tokens used to generate this artifact. |
| **Copy** | Copies the document `content` to the clipboard as plain text. |
| **Download Excel** | Exports the artifact as an `.xls` file (see [section 4](#4-download-excel)). |
| **Edit** | Toggles the Edit panel on the right (see [section 5](#5-edit-and-regenerate)). |
| **Regenerate** | Re-runs the original request and replaces the artifact in place. |

### Document tab

The main narrative of the artifact, rendered as formatted markdown — headings, bold text,
tables, lists, and paragraphs. This is the `content` the LLM wrote based on your description
and the live query results.

### Data tab

Appears only when the artifact contains at least one result set with rows. If there are
multiple datasets (the agent may run several sub-queries for a complex document), a **tab
bar** lets you switch between them. Each dataset shows a scrollable data table of its rows.

---

## 4. Download Excel

Click **Download Excel** to export the artifact as a `.xls` spreadsheet. The file is
assembled entirely in the browser — no upload required.

The workbook contains three sheet types:

| Sheet | Content |
|---|---|
| **Report** | The document narrative converted to a worksheet. Headings appear as bold blue-fill cells; markdown tables become native worksheet rows with numeric type detection. |
| **Data (1, 2, …)** | One sheet per dataset — the raw rows returned by the SQL queries, with auto-detected column headers. |
| **SQL** | All SQL queries that the agent ran, one per section. |

The file is downloaded immediately with the artifact name as the filename.

---

## 5. Edit and regenerate

Click **Edit** in the header to open the Edit panel alongside the document.

| Field | What you can change |
|---|---|
| **Name** | Rename the artifact. |
| **Format** | Switch the document type (dropdown). |
| **Purpose** | Rewrite the audience / decision context (step 2 of the wizard). |
| **Data Focus** | Rewrite the data description (step 3 of the wizard). |
| **SQL Override** | A terminal-style textarea pre-filled with the first dataset's SQL. Edit it to lock in a specific query instead of letting the agent re-derive it. Leave unchanged to let the agent regenerate freely. |

Click **Apply & Regenerate** to submit. The agent re-runs the request with your updated
values and replaces the artifact in place. If the SQL Override is unchanged from the
original, the agent regenerates from the purpose and data focus only. If the SQL Override
is edited, the agent uses your SQL directly and skips query planning for the first dataset.

> **Tip —** the SQL Override is the most powerful edit. If the agent's original SQL was
> almost right — wrong filter, wrong date range — paste a corrected version here and
> regenerate; the narrative updates to match the new data.

---

## 6. Regenerate without editing

Click **Regenerate** in the header (without opening the Edit panel) to re-run the exact
original request — same name, purpose, data focus, format. The artifact is replaced in place.
Use this when the underlying data has changed and you want a fresh document.
